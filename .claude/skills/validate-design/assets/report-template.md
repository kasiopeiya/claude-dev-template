# Design Validation Report Template

検証対象の設計書は、プロジェクトの構成に応じて 1 件または複数件になる。各設計書ごとに結果セクションを作る。

## すべて整合の場合

```
=== Design Validation Report ===

検証対象:
  - {設計書パス}

検証結果: ✅ すべて整合

検証項目:
  ✓ {カテゴリ}: {検証項目名} ({整合数}/{総数})
  ✓ ...

すべての検証項目で整合性が確認されました。
```

## 不整合を検出した場合

```
=== Design Validation Report ===

検証対象:
  - {設計書パス}

検証結果: ⚠️ 不整合を検出

不整合の詳細:

📝 {カテゴリ}: {検証項目名}

{連番}. {問題の説明} [{ERROR|WARNING}]
   - {対象種別}: {名前/値}
   - 設計書: {パス} ({見出し参照})
   - 実装: {ファイルパス}
   - 推奨対応: {修正内容}

---

整合性のある項目:
  ✓ {カテゴリ}: {検証項目名} ({整合数}/{総数})
  ✗ {カテゴリ}: {検証項目名} ({整合数}/{総数} - {N}件不整合)
  ⚠ {カテゴリ}: {検証項目名} ({整合数}/{総数} - {N}件警告)

推奨アクション:
{連番}. {修正内容}
```

## 複数の設計書を検証した場合（サマリー付き）

```
=== Design Validation Report ===

検証対象:
  - {設計書パス A}
  - {設計書パス B}
  - ...

検証結果: ⚠️ 不整合を検出  ← または ✅ すべて整合

サマリー:
  - {設計書パス A}: {N}件の不整合  ← または「整合性あり」
  - {設計書パス B}: {N}件の不整合 ← または「整合性あり」

---

## {設計書パス A} の検証結果

{上記の「不整合を検出した場合」または「すべて整合の場合」の形式を適用}

---

## {設計書パス B} の検証結果

{上記の「不整合を検出した場合」または「すべて整合の場合」の形式を適用}
```
