# CDK設計ポリシー


# 目的

- 実装するときとレビューのチェックリストとして使う予定
- 書き方のレベルの話はcdk.mdに書く。ここに書くのは設計の話のみ。


# 書く予定の内容

- 環境差分の実装方法
- 外部依存パラメータはparameter.tsを見ればわかるようにする
- リソース名はなるべく固定しない
　固定する必要があるときはStack名をプレフィックスにし、固定する理由をコメントに書く
- Stackの実装は抽象的にする
　Stackのconstructorは全体を鳥瞰できるような可読性の良い実装にする
　リソースを大量に実装する場合、全てをStackのレイヤーに書いていると見通しがわるくなる
　抽象化のための対策　
    - L1 Constructなどは自作のConstructを作って抽象化する
    - Stackや自作Constructのメソッドを使って処理をまとめる
